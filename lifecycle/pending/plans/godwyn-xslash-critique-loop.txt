### Problem statement
We want an automated loop that iterates specifically until Godwyn's X-slash reads as a
GOOD X-slash — measurably, not just "not broken." The loop machinery exists; what is
missing is (a) a checkable definition of good, (b) X-shape-aware measurement, and (c) a
driver that points the existing loop + describe-swarm at those X targets and threads the
critique back to Fable so it converges on the X instead of spinning on generic polish.

### Success criteria (measurable) — THE "GOOD X-SLASH" ACCEPTANCE SPEC
The driver run is ACCEPTED when the final animation satisfies ALL of the below. These are
the load-bearing new definition of "good"; §4 gives the exact measurement.
  S0. GENERIC BASELINE (from the parent, unchanged): anim_metrics.py PASSES all four hard
      families — no cape/robe explosion (M1), no body contortion/jerk (M2), no foot-slide
      while planted (M3), no sword detach (M4). This is necessary but NOT sufficient.
  S1. TWO DISTINCT DIAGONAL CUTS. The blade-tip world path over the clip decomposes into
      exactly TWO dominant sweeping strokes, temporally separated (cut1 completes, then
      cut2 begins — not one smear). (X1 in §4.)
  S2. CORRECT DIAGONAL DIRECTIONS + ORDER. Cut1 is a \ (starts upper-right, ends
      lower-left); cut2 is a / (starts upper-left, ends lower-right); cut1 precedes cut2.
      Each stroke's dominant direction vector matches the target diagonal within a
      tolerance cone. (X2 in §4.)
  S3. THE PATHS ACTUALLY CROSS — A VISIBLE X. The two blade-tip stroke paths, projected
      onto the frontal (camera-facing) plane, INTERSECT roughly in front of the torso
      (crossing point within the mid-chest region, not off to one side, not behind).
      (X3 in §4.)
  S4. SPINE/SHOULDER-DRIVEN POWER. The spine + shoulder bones lead the cut (their angular
      motion peaks BEFORE the wrist/hand's), not an arm-only flail. (X4 in §4.)
  S5. SPIN CAP. Total root/hips yaw rotation rate never exceeds ~1 rotation/sec
      (360°/sec) at any point — SPEC §7 user-calibrated. (X5 in §4.)
  S6. CAPE WHIP-THEN-SETTLE. The cape/robe phys_ chains show a WHIP (elevated but
      NON-exploding motion — M1 still passes) DURING each cut, followed by a SETTLE
      (motion decays toward rest) after the second cut. Not frozen-stiff, not exploding.
      (X6 in §4.)
  S7. WEIGHT-SHIFT + FOOT PLANT. A readable weight-shift (hips CoM translates) with the
      cuts and a small forward step; feet stay planted during contact (M3), i.e. the step
      is a deliberate replant, not a slide. (X7 in §4.)
  S8. HUMAN FEEL (final judge, non-automatable). The human watches renders/game/
      godwyn_xslash_v2.mp4 and confirms it reads as Godwyn's signature X. The driver
      NEVER declares final victory without surfacing awaitingUser (parent already does).
  S9. The describe-swarm critic was ACTUALLY USED each round via the child workflow, its
      fixBrief[] was threaded to Fable, and the X-acceptance evaluation (S1-S7) is logged
      per round so convergence on the X is visible in the audit trail.

             animName:'Godwyn_XSlash',

             probeAspects: <X-tuned describe aspects>,  // NEW arg the parent forwards to child
             vlmModel, maxRounds, patience, ... })
         │
         ▼   inside the engine (UNCHANGED except the additive P2.B hook):
   ┌───────────────────────────────────────────────────────────────────────────┐
   │ P1 Fable build (X-slash)                                          [fable]  │
   │ P2 SERIAL loop, each round:                                                │
   │   P2.A anim_metrics.py  -> generic families M1..M4 (G-4 gate)     [sonnet] │
   │   P2.A+ anim_metrics_xslash.py -> X1..X7 (NEW add-on, additive)   [sonnet] │  <-- D2
   │   P2.B workflow('godwyn-vlm-critic', { ..., probeAspects,                  │
   │         metricFlaws (generic + X) }) -> describe-swarm -> fixBrief[]       │  <-- child
   │        the X-tuned aspects steer the swarm at cape whip/settle,            │
   │        swing snap, cross geometry, weight-shift, follow-through            │
   │   G-4 combine: roundPass = M1..M4 PASS AND X1..X7 PASS AND critic-abstain- │  <-- D1 acceptance
   │        or-no-corroborated-blocker. X failures become metric-grade flaws.   │
   │   P2.C Fable fixer consumes fixBrief[] (X-anchored first)         [fable]  │
   │   checkpoint each round (parent, unchanged)                                │
   │ P3 export godwyn_xslash_v2.glb + mp4 (parent, unchanged)          [sonnet] │
   └───────────────────────────────────────────────────────────────────────────┘
         │
         ▼

         emit the acceptance report + awaitingUser (watch the mp4).             [sonnet]

## 4. THE X-SLASH MEASUREMENT SPEC — scripts/anim_metrics_xslash.py (X1..X7)
=====================================================================

All quantities are computed per frame by stepping scene.frame_set across the action range
(Blender 5.2 safe — NO action.fcurves), normalized by character_height (rest-pose body
bbox Z). Inputs from P1: --tip (blade-tip world point spec), --cam-forward (frontal-plane
normal). Thresholds live in one top-of-file dict (easy re-tune); --calibrate grounds them.

  X1  TWO DISTINCT CUTS  (S1)
    Compute blade-tip speed per frame (smoothed). Segment the clip on speed minima into
    strokes. FAIL if the clip does not decompose into exactly TWO dominant high-speed
    strokes separated by a clear low-speed trough (i.e. one smear, or three+ swings, is not
    the X opener). PASS = 2 strokes, cut1 then cut2, with a discernible gap.
    Threshold: gap trough speed < 0.35 * peak stroke speed; each stroke arc-length >
    0.5·character_height.

  X2  DIAGONAL DIRECTIONS + ORDER  (S2)
    For each stroke, the dominant travel vector (tip end minus tip start), projected onto
    the frontal plane. cut1 must be a \ (start upper-right, end lower-left): its projected
    direction within a tolerance cone of the target down-left diagonal. cut2 must be a /
    (start upper-left, end lower-right): within the cone of the down-right diagonal. cut1
    must temporally precede cut2.
    Threshold: each stroke direction within 30° of its target diagonal; ordering \ then /.

  X3  VISIBLE CROSSING X  (S3)  — the signature test
    Project both stroke tip-paths onto the frontal plane. Compute their intersection point.
    FAIL if the paths do not intersect, OR the intersection lies outside the mid-torso
    region (a box centered on the chest bone, +/- ~0.25·character_height in each frontal
    axis), OR the crossing is behind the body. PASS = a single clean crossing in front of
    the mid-chest — the visible X.
    Threshold: crossing exists AND within the mid-chest box AND on the front (camera) side.

  X4  SPINE/SHOULDER-DRIVEN POWER  (S4)
    Angular-velocity peak timing: the spine + clavicle/shoulder bones' peak angular speed
    must occur BEFORE the RightHand/wrist peak in each cut (power initiates from the core,
    the hand follows). FAIL if the hand leads (arm-only flail) in either cut.
    Threshold: shoulder/spine peak frame <= hand peak frame per cut (small tolerance
    +/- 2 frames); core angular contribution >= a floor fraction of total.

  X5  SPIN CAP  (S5)
    Root/hips yaw rate (Δ yaw / Δt) across the clip. FAIL if any frame exceeds 360°/sec
    (1 rotation/sec, SPEC §7 user-calibrated). PASS = weighty, readable, never whirling.
    Threshold: max hips yaw rate <= 360°/sec.

  X6  CAPE WHIP-THEN-SETTLE  (S6)
    For phys_cape/robe verts: motion energy (per-frame displacement magnitude) must RISE
    during each cut (a whip) and DECAY after cut2 toward rest (a settle) — while STILL
    passing M1 (no explosion). FAIL if the cape is frozen-stiff (no rise during cuts:
    energy variance below a floor => lifeless) OR never settles (energy still high N frames
    after cut2 => flailing). NOTE: M1 explosion is a SEPARATE, harder fail owned by
    anim_metrics.py; X6 is the "alive but controlled" band between stiff and exploding.
    Threshold: per-cut cape energy peak > 1.5× rest-idle energy (whip present) AND
    post-cut2 energy decays to < 1.3× rest within ~0.4s (settle) AND M1 still PASS.

  X7  WEIGHT-SHIFT + FOOT PLANT  (S7)
    Hips center-of-mass horizontal translation must show a readable shift with the cuts +
    a small net forward step (SPEC §7 "feet planted then a small forward step"), while each
    planted foot stays planted during contact (defers to M3 for slide). FAIL if the hips
    are static (no weight-shift => floaty upper-body-only) OR a foot slides during contact
    (M3, cross-checked).
    Threshold: hips horizontal CoM travel between 0.05·h and 0.5·h (a shift, not a lunge)
    AND net forward step > 0.03·h AND M3 PASS (no contact-slide).

OUTPUT JSON:
  {
    animName, action, frameRange, characterHeight, tipSpec, camForward,
    xFamilies: {
      X1_twoCuts:   { pass, strokeCount, gapTroughRatio, arcLengths },
      X2_diagonals: { pass, cut1DirDeg, cut2DirDeg, order },
      X3_crossing:  { pass, crossingPoint, inMidChest, front },
      X4_power:     { pass, cut1: {shoulderPeakF, handPeakF}, cut2: {...} },
      X5_spin:      { pass, maxYawRateDegPerSec },
      X6_cape:      { pass, whipPeakRatio, settleTime, m1Pass },
      X7_weight:    { pass, hipsCoMTravel, netForwardStep, m3Pass }
    },
    overallXAccept,                 // AND of X1..X7
    xFlaws: [ { criterion, whatWrong, howToFix, severity } ]  // prioritized, Fable-phrased
  }
Idempotent; writes to /tmp; prints path. --calibrate prints a suggested-threshold block
from a good-X / non-X envelope. If calibrate disagrees with the defaults by >2x, RECORD
it for the user (OQ-2), proceed with the stricter.
