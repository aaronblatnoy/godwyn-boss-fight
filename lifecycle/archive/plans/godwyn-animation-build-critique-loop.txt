## 0. CONTEXT & GOALS

### Problem statement
We need to author Godwyn's moveset animations (X-slash first, then any move via args)
on the rigged models/godwyn_game.glb, and we need an AUTOMATED quality gate that
catches the failure modes that actually happen here: exploding cape/robe spring
chains, contorted/jerky bodies, feet sliding, sword detaching from the hand. The
obvious design — "render a flipbook, ask a VLM PASS/FAIL" — is UNSAFE:
  - qwen2.5vl:7b is effectively BLIND for this task. On a KNOWN-BAD animation (cape
    exploding, body contorted) it returned "cape flows naturally, body coherent,
    PASS 9/10", and on a neutral describe-prompt it hallucinated the scene AND
    miscounted the grid (called a 4x4/16-panel sheet "3x4/12"). It rides prompt
    priors, it does not reliably read pixels.
  - qwen2.5vl:32b MIGHT genuinely see (a neutral-prompt test on the same known-bad
    sheet was RUNNING at plan-writing time — OUTCOME UNKNOWN, see §10 OQ-1), but at
    ~0.19 t/s it is 20–40 min per critique AND pins the box at max load. black-sky
    HARD-LOCKED under load on 2026-07-18 (memory blacksky-hardlock-2026-07-18). So
    even a seeing 32B cannot be the loop's fast, always-on gate.

Therefore a VLM PASS must NEVER be trusted on its own to greenlight an animation.

### Success criteria (measurable)
1. Running the workflow with no args authors the default X-slash and produces:
   models/<anim>_v2.glb (rig+skin+baked-materials+action, verified by re-import) and
   renders/game/<anim>_v2.mp4 for the human to watch.
2. Every candidate animation is gated by an OBJECTIVE metrics report (§4 script
   scripts/anim_metrics.py) computed directly from the Blender action — no VLM in
   the reliable path. The metrics are hard thresholds (§ P2.A) that flag explosion,
   contortion/jerk, foot-slide, and sword-detach; these CANNOT hallucinate.
3. The VLM is a SECONDARY judge only, model-configurable (7b default fast / 32b
   opt-in slow), and CANNOT override a metrics FAIL (metrics win — hard rule G-4).
4. The VLM path carries a built-in ADVERSARIAL SANITY probe (a neutral panel-count
   question the agent verifies) so a blind model is DETECTED and its verdict
   DOWN-WEIGHTED, not trusted.
5. The loop is convergence-aware: a max-round cap AND plateau/patience detection on
   a COMBINED score, so it stops instead of spinning; it halts safely (partial
   result, never null-cascade) if any phase returns null.
6. Rig, skinning, and baked materials are byte-preserved across every edit; the FACE
   is never touched; cape/robe stay damped-spring bone chains (never full cloth sim).
7. Exported + committed local AND server; NEVER pushed.

### Non-goals (explicitly out of scope)
- Godot import / in-engine hitbox wiring (that is godot-combat-foundation-plan.txt).
- Authoring the full moveset tree in one run (this builds ONE anim per invocation;
  reusability = re-invoke with different args).
- Making the 7B VLM reliable (we treat it as untrusted by design).
- Replacing the human as final judge of FEEL — the human always watches the mp4.
- Mocap retarget authoring (mocap_retarget.py exists; this plan's default is
  keyframe authoring, but P1 MAY seed from an existing mocap glb if args say so).

### Assumptions made
- A-1: Rig facts confirmed from scripts (robe_bones.py, anim_xslash_cape.py): the
  physics chains are bone chains prefixed `phys_` (e.g. phys_cape_C_00, plus
  phys_robe_*, phys_hair_*), parented Hips/Spine/Head; body is the 24-bone Mixamo
  set; the sword is object `Godwyn_Sword` parented to bone `RightHand`. The metrics
  script keys off these prefixes. P0 RE-VERIFIES them against the actual glb and
  fails the gate if they've drifted (do not hardcode blindly).
- A-2: Blender 5.2 slotted actions — action.fcurves is GONE. Authoring uses
  pose_bone.keyframe_insert. Metrics extraction reads bone matrices per-frame via
  scene frame-stepping + matrix_world / matrix_channel, NOT action.fcurves.
- A-3: EEVEE (BLENDER_EEVEE enum) for all preview/flipbook/mp4 renders (fast);
  Cycles+OptiX only if a beauty still is ever needed (not required by this loop).
- A-4: ffmpeg is on black-sky (confirmed used by prior workflows).
- A-5: The default target is the X-slash (SPEC §7 "author first" move). Args can
  override to any move; the animDesc string is passed to Fable and the VLM verbatim.
- A-6: The metrics thresholds below are STARTING values calibrated to be caught by
  the known-bad references already on the box (the exploding-cape sheets in
  /tmp/godwyn_cloth_r*); P2.A includes a self-calibration sub-step that measures a
  known-good and known-bad clip once and prints suggested thresholds, so the numbers
  are grounded, not guessed. If self-calibration and these defaults disagree by >2x,
  the executor surfaces it (§10 OQ-2) rather than silently trusting either.

## 1. ARCHITECTURE OVERVIEW

This is a Workflow (JS orchestration) that fans agents out over `ssh black-sky`.
No app services / DB / queues — the "invariants" here are the Godwyn build rules.

Data flow (one loop iteration):
                                            ┌─────────────────────────────┐
  Fable authors/edits  scripts/anim_<m>.py │  RELIABLE BACKBONE GATE      │
        │  (keyframes on pose bones)        │                             │
        ▼                                    │  scripts/anim_metrics.py    │
  headless bake → WIP animated .blend/.glb ─┤  reads the action per-frame │
        │                                    │  → JSON metrics report      │
        ├──► render 2 flipbooks (front+3/4) ─┤  cape/robe max-displacement │
        │    (ffmpeg contact sheets)         │  bone ang-vel + jerk        │
        │            │                        │  foot horizontal slide      │
        │            ▼                        │  sword-hand distance        │
        │     @vlm-critic (configurable VLM) │  → PASS/FAIL vs thresholds  │
        │     SECONDARY subjective read +    └──────────────┬──────────────┘
        │     adversarial sanity probe                      │
        │            │                                       │
        └────────────┴──────────► COMBINE (G-4: metrics win) ▼
                                     │ pass? → export (P3)
                                     │ fail? → Fable FIXER consumes prioritized flaws
                                     │          (metrics flaws first, VLM flaws second)
                                     └──────► loop (cap + plateau on combined score)

Affected "services": all work is Blender-on-black-sky (author local → scp → headless).

### Invariants this plan MUST preserve (from CLAUDE.md + measured facts)
- INV-1 Headless-only on black-sky (ssh "cmd", no -t, no GUI). Author scripts LOCALLY,
  scp, run headless. Write previews/sheets to /tmp on server (NEVER ~/Desktop).
- INV-2 GPU-real: assert OptiX/CUDA in any Cycles render; EEVEE for previews is fine.
- INV-3 Idempotent bpy scripts (delete-by-name before recreate).
- INV-4 Never auto-push. Commit local + server after phases; push only on explicit ask.
- INV-5 Preserve armature + skinning + baked image-texture materials on EVERY edit;
  do NOT rebuild UVs/materials; do NOT touch FACE geometry.
- INV-6 Cape/robe/hair = STABLE damped-spring `phys_` bone chains. NEVER a full cloth
  sim (it explodes — see the 13 rounds of /tmp/godwyn_cloth_r* history). If metrics/VLM
  report explosion → damp harder / lower stiffness / clamp per-frame delta, do NOT
  switch to an unstable sim.
- INV-7 black-sky stability: NEVER run two VLM calls in parallel; the 32B is a
  heavy-load profile and this box has hard-locked under load. VLM calls are serialized,
  one at a time, and the loop must not block indefinitely on a 32B call (timeout + guard).
- INV-8 SPEC-fidelity: motion vocabulary (X-slash crossing cuts, spin ≤1 rot/sec,
  toes forward not splayed, combat grip) comes from SPEC §7; do not invent timings.

### PHASE 2: Critique loop — Metrics gate (backbone) + VLM cross-check → Fable fixer
Prerequisites: P1 (WIP glb + action).
This is the heart of the plan. It is a bounded, convergence-aware loop. Each round:
  (A) METRICS — the RELIABLE gate; (B) VLM — SECONDARY, sanity-guarded; combine (G-4);
  if fail, (C) Fable FIXER consumes prioritized flaws; re-loop.

  #### P2.A — OBJECTIVE METRICS (the backbone gate) — subagent_type general-purpose, model sonnet
  Run scripts/anim_metrics.py on the current WIP glb/action → JSON. Metrics + thresholds
  (all computed per-frame across the clip; a family FAILS if any frame breaches):
    M1 CAPE/ROBE EXPLOSION — max per-frame vertex displacement of phys_cape_*/phys_robe_*
       driven verts, in rig-local units, AND max per-frame growth of the cloth
       bounding-box vs its rest size.
         FAIL if max single-frame vertex delta > 0.06 * character_height
         OR bbox_frame / bbox_rest > 1.8  (explosion/ballooning).
    M2 BODY CONTORTION / JERK — for each body bone: angular velocity (Δangle/frame) and
       jerk (Δ²angle/frame²) of the bone's local rotation.
         FAIL if any body-bone angular velocity > 45°/frame (teleport/pop)
         OR jerk > 30°/frame²  (jerky snap) on more than 1 isolated frame
         OR any joint exceeds an anatomical rotation limit (elbow/knee hyperextension
            > 175°, spine twist > 90° between adjacent spine bones).
    M3 FOOT SLIDE — for each planted foot (contact = toe/heel bone world-Z below a
       ground epsilon), horizontal (XY) world displacement between consecutive
       contact frames.
         FAIL if a foot in contact slides horizontally > 0.02 * character_height/frame
         (i.e. it should be planted, not skating).
    M4 SWORD DETACH — world distance between Godwyn_Sword grip point and the RightHand
       bone head across the clip.
         FAIL if distance > 1.15 * rest_grip_distance on any frame (blade left the hand).
    M5 (advisory, not a hard gate) SMOOTHNESS SCORE — 10 minus a normalized penalty from
       M2 jerk integral, reported 0–10 for the combined-score trajectory.
  Output: per-family PASS/FAIL, the worst frame + value per family, and a prioritized
  metricFlaws[] (issue, concrete fix, severity) a fixer can act on directly, e.g.
  "M1 cape explodes at f042 (Δ0.11 > 0.06·h): increase phys_cape damping / lower
  stiffness / tighten per-frame delta clamp."
  CALIBRATION (once, in round 1 only): also run --calibrate against a known-good clip
  (e.g. models/godwyn_xslash.glb if it reads clean) and a known-bad reference; print the
  measured good/bad envelope so the thresholds above are grounded. If measured envelope
  disagrees with the defaults by >2x, RECORD it for the user (OQ-2) but PROCEED using the
  stricter of the two.

  #### P2.B — VLM SUBJECTIVE CROSS-CHECK — subagent_type general-purpose (drives @vlm-critic), model sonnet
  Only invoked if metrics PASS, OR to enrich a metrics FAIL with visual specifics
  (configurable; default = run it every round so its trajectory is logged, but it is
  NEVER load-bearing on a metrics FAIL). Steps:
    1. scripts/anim_flipbook.py renders 12–16 evenly-spaced EEVEE frames from front +
       3/4 side, ffmpeg-tiles each into ONE labeled contact sheet on the server (/tmp).
    2. ADVERSARIAL SANITY FIRST: ask the configured VLM the neutral panel-count probe on
       the actual sheet ("How many panels are in this grid? single integer"). If wrong →
       mark vlmTrust='blind' for THIS round and DOWN-WEIGHT its verdict to zero (metrics
       fully decide). This detects a blind model per-round, not just at P0.
    3. Then the motion critique: this is a CONTACT SHEET / flipbook read L-to-R, T-to-B;
       Godwyn (golden demigod swordsman, royal-blue cloth) performing <animDesc>; judge
       smoothness / cape-explosion / contortion / foot-slide / reads-as-move; end with
       VERDICT PASS|FAIL + SMOOTHNESS X/10 + concrete per-frame flaws. Relay VERBATIM.
    4. Model policy: vlmModel default '7b' (fast, treated as suspect); '32b' only if
       args opt in — and even then run ONE call at a time, generous timeout (up to
       10 min), and if it exceeds the timeout, record vlm='timeout' and let METRICS
       decide the round (never block the loop on a hung 32B — INV-7).
  Output: VLM_CRITIQUE (pass, smoothness, flaws[], vlmVerdictRaw, vlmTrust-this-round).

  #### G-4 COMBINE (the hard rule):
    - roundPass = metricsPass AND (vlmTrust=='blind' ? true : vlmPass)
        i.e. metrics are NECESSARY; a trusted VLM PASS is also required, but a
        BLIND/timeout VLM cannot BLOCK a metrics PASS (it just abstains).
    - If metrics FAIL → roundPass=false REGARDLESS of a VLM PASS. Metrics win. Always.
    - combinedScore = min(metricM5_smoothness, (vlmTrust=='blind'?10:vlmSmoothness))
        used ONLY for plateau detection, not for pass/fail.

  #### P2.C — FABLE FIXER (only if !roundPass and not last round and not plateaued)
  subagent_type general-purpose, model fable. Consume flaws in priority order:
  METRIC flaws FIRST (objective, unambiguous), THEN trusted-VLM flaws. Edit
  scripts/anim_build_<anim>.py (keep rig/materials/chains/face intact; cape stays
  damped-spring; clamp harder if M1 fired). Re-run headless, re-export WIP glb, quick
  self-check + READ. Return ok + changes.

  #### CONVERGENCE / LOOP CONTROL
    - MAX_ROUNDS = 5 (default; args.maxRounds override).
    - PATIENCE = 2 on combinedScore: if combinedScore does not IMPROVE for 2
      consecutive rounds, STOP (plateau) and exit the loop with the best WIP so far.
    - Stop immediately on roundPass=true (success) OR if the fixer returns null/!ok
      (cannot apply — exit with best-so-far, mark needsUserJudgment).
    - Every round appends to critLog: {round, metrics JSON path, metricsPass,
      per-family worst values, vlmPass, vlmSmoothness, vlmTrust, combinedScore,
      contactSheets, fixerChanges}.

Validation (phase-level):
  - Loop terminates within MAX_ROUNDS with a definite state: passed | plateaued |
    fixer-stuck. The final WIP glb has a metrics JSON attached. If it never metrics-PASSes,
    the phase still EXITS cleanly (does not hang) and flags needsUserJudgment.
Risks / Edge cases:
  - Metrics false-positive (threshold too strict) → the calibration sub-step + reporting
    the worst-frame value lets the user re-tune; the loop won't silently spin because
    plateau detection stops it.
  - 32B hangs / box under load → timeout + abstain + metrics decide (INV-7); never a
    second concurrent VLM call.
  - VLM says PASS on a broken anim (the exact known failure) → G-4 makes metrics
    authoritative; a metrics FAIL overrides. This is the core robustness guarantee.
  - Fixer over-damps cape into rigidity → M5 smoothness + VLM "stiff cape" flag catch it;
    the fixer brief says damp ENOUGH to stop explosion, not to freeze.
Subagent orchestration (loop body — the executor emits these per round):
  - P2.A METRICS: [SINGLE] general-purpose, model sonnet, background:no
      Brief: "Round R of Phase 2 of <PLAN>. Run scripts/anim_metrics.py on <WIP_GLB>
       action <ANIM>. Read §4 + P2.A thresholds. (Round 1 also: --calibrate.) Return
       per-family PASS/FAIL, worst frame+value, prioritized metricFlaws[]. Touch nothing
       but /tmp + read. Print one-line summary + exit."
  - P2.B VLM: [SINGLE] general-purpose driving @vlm-critic, model sonnet, background:no
      (32b rounds are >2min but must gate the round, so still awaited with a timeout;
       do NOT background because the loop needs the verdict — but NEVER run two at once.)
      Brief: "Round R of Phase 2 of <PLAN>. Read P2.B. Build front+3/4 flipbooks via
       scripts/anim_flipbook.py from <WIP_GLB>. Run the panel-count sanity probe FIRST
       (set vlmTrust). Then the motion critique with VLM <VLM_MODEL> (one call at a time,
       10-min timeout; on timeout return vlm='timeout'). Return pass, smoothness, flaws[],
       vlmVerdictRaw, vlmTrust. Print one-line summary + exit."
  - P2.C FIXER: [SEQUENTIAL — depends on that round's A+B] general-purpose, model fable,
      background:no
      Brief: "Round R fixer of Phase 2 of <PLAN>. Read INV-5/6. Fix these flaws
       (metrics first, then VLM): <PRIORITIZED_FLAWS>. Edit scripts/anim_build_<anim>.py,
       re-run headless, re-export WIP glb, self-check + READ. Keep rig/materials/face/
       damped-spring intact. Return ok + changes. Commit (no push). One-line summary + exit."
  Note: A→B→C are strictly SEQUENTIAL within a round (shared WIP glb file + serialized
  VLM). Rounds are sequential (each depends on the prior fix). NO parallel fanout here —
  this is an inherently serial refine loop (see §12.2 shared-file serialization).

### PHASE 3: Export + commit + human-watch mp4
Prerequisites: P2 (best WIP glb, passed or best-effort).
Deliverables:
  - Final animated glb models/<anim>_v2.glb via glTF: armature (Mixamo + phys_ chains)

## 4. DATA MODEL CHANGES — scripts/anim_metrics.py spec (the objective backbone)

No DB. This is the reusable measurement script — the load-bearing new artifact.

INPUT: an animated .glb or .blend + action name (+ optional thresholds JSON + optional
--calibrate good.glb bad.glb). Reads bone world matrices per frame by stepping
scene.frame_set across the action range (Blender 5.2 safe — no action.fcurves).

COMPUTES per frame, then reduces across the clip:
  - character_height: rest-pose bbox Z of the body mesh (normalizer for all thresholds).
  - CAPE/ROBE displacement: for verts weighted to phys_cape_*/phys_robe_* (prefix is a
    PARAM from P0 rigMap, not a constant): max per-frame vertex delta; cloth-bbox growth
    ratio vs rest.  → M1
  - BODY bone angular velocity + jerk: per body bone, Δ and Δ² of local Euler/quat angle
    between frames; plus anatomical-limit checks on elbow/knee/spine bones. → M2
  - FOOT slide: detect contact frames (toe/heel world-Z < ground_eps); horizontal XY
    world delta between consecutive contact frames per foot. → M3
  - SWORD detach: world distance Godwyn_Sword grip ↔ RightHand head, vs rest. → M4
  - SMOOTHNESS (M5): 10 − normalized jerk integral (advisory).

OUTPUT JSON:
  {
    animName, action, frameRange, characterHeight,
    families: {
      M1_cape:   { pass, worstFrame, worstDelta, bboxGrowth, threshold },
      M2_body:   { pass, worstFrame, worstAngVel, worstJerk, limitViolations[] },
      M3_foot:   { pass, worstFrame, worstSlide, foot },
      M4_sword:  { pass, worstFrame, worstDistRatio }
    },
    overallMetricsPass,          // AND of the four hard families
    smoothnessM5,                // 0–10 advisory
    metricFlaws: [ { issue, fix, severity } ]   // prioritized, fixer-consumable
  }
Idempotent, writes to /tmp; prints the JSON path. --calibrate prints a suggested-threshold
block from the good/bad envelope. Thresholds live in one dict at the top (easy re-tune).

THRESHOLD DEFAULTS (from §P2.A; ×character_height where noted):
  M1 vertex delta 0.06·h ; cloth bbox growth 1.8× rest
  M2 ang-vel 45°/frame ; jerk 30°/frame² (>1 frame) ; elbow/knee <175° ; spine-twist <90°
  M3 foot slide 0.02·h per frame while in contact
  M4 grip distance 1.15× rest

## 8. TESTING STRATEGY
- Metrics-tool unit sanity (P0): run anim_metrics.py on a KNOWN-GOOD and a KNOWN-BAD clip
  already on the box; assert it FAILS the bad (cape explosion should trip M1) and PASSes
  the good. This is the single most important test — it proves the backbone actually
  discriminates before we trust it.
- VLM adversarial probe (P0 + every P2.B round): panel-count question; a wrong count
  proves the model is blind for THIS sheet → down-weight to abstain.
- Loop-termination test: force a permanently-failing metric (temp strict threshold) and
  confirm the loop stops at maxRounds/plateau, not spins.
- Export re-import test (P3): assert action track + phys_ chains + skin + textures + sword.
- Manual QA: human watches the mp4 (feel); spot-reads the final contact sheets.

## 12. EXECUTOR ORCHESTRATION GUIDE

### 12.1 Parallelism map
  Phase | Execution   | Siblings | Depends on | Model  | Subagent type
  P0    | SINGLE      | —        | none       | sonnet | general-purpose
  P1    | SEQUENTIAL  | —        | P0         | fable  | general-purpose
  P2.A  | SINGLE      | —        | P1 / prevC | sonnet | general-purpose  (metrics)
  P2.B  | SINGLE      | —        | P2.A       | sonnet | general-purpose  (drives @vlm-critic)
  P2.C  | SEQUENTIAL  | —        | P2.A+P2.B  | fable  | general-purpose  (fixer)
  P3    | SEQUENTIAL  | —        | P2         | sonnet | general-purpose

  DAG (linear — this is an inherently serial refine loop; no fanout):
    P0 → P1 → [ P2.A → P2.B → P2.C ]×(1..maxRounds, until pass/plateau) → P3

  Note on subagent_type: no specialized agent (refractor-pair / audit-duo /
  consensus-verification-duo) is invoked. The one project-specific asset, @vlm-critic,
  is a Claude(sonnet) agent DRIVEN from P2.B — the executor may either dispatch a
  general-purpose agent that calls @vlm-critic, or dispatch @vlm-critic directly as the
  P2.B agent (it exists at .claude/agents/vlm-critic.md and is the right role). Either is
  fine; do NOT inline-personate the VLM read (the whole point is the real client.py call).

### 12.2 Cross-phase work consolidation / shared-file serialization
  - The WIP animated glb + scripts/anim_build_<anim>.py are edited by P1 and every P2.C
    fixer round. These MUST be serialized — that is why P2 is a serial loop, not a fanout.
    Never run two fixer rounds concurrently (Edit/file conflict + they'd fight the same glb).
  - anim_metrics.py + anim_flipbook.py are authored ONCE in P0 and only READ thereafter —
    stable, no serialization concern.
  - The VLM path is serialized by INV-7 regardless of file concerns (box stability).

### 12.3 Model-mix budget
  Rough split across dispatched agents (a 3-round run ≈ P0 + P1 + 3×(A,B,C) + P3 = 12 agents):
    fable  ≈ 33%  (P1 + 3 fixer rounds — the artistic craft; genuinely needs top motion feel)
    sonnet ≈ 67%  (P0, all metrics, all VLM-driver, export — mechanical)
    opus    0%   (nothing here is a schema-affecting/multi-file-reasoning refactor)
    haiku   0%   (nothing is a trivial single-field edit)
    fable   0 for the "frontier" tier beyond the artistic build — no whole-system proof phase.
  This matches the project rule: Fable = artistic build+fixes; Sonnet = setup/render/
  metrics/export; VLM (via @vlm-critic) = subjective. Sanity-check: fable only on P1 + fixers.

### 12.4 Estimated wall-clock
  Best case (7b VLM, passes round 1): P0(~15m) + P1(~15m) + 1×[A(~2m)+B(~3m)] + P3(~8m)
    ≈ 45 min.
  Typical (7b, 3 rounds): ≈ 75–90 min.
  Worst case with 32b opt-in (3 rounds, 30 min/VLM call): dominated by 3×~30m VLM
    ≈ 2.5–3.5 hours AND box-hardlock risk (OQ-4). This is exactly why 32b is opt-in and the
    loop NEVER blocks indefinitely on it (timeout → metrics decide).
  Critical path: the P2 loop; each round is serial (A→B→C) and rounds are serial.

### 12.5 Failure recovery policy
  - P0 (rig verify OR metrics-tool smoke fails): PAUSE + escalate — the whole gate depends
    on it; do not proceed with a broken backbone. (This is a hard stop.)
  - P1 (build fails/null): retry once with same fable subagent; if still null, escalate.
  - P2.A metrics agent null: retry once; if still null, HALT the loop (no backbone → cannot
    safely gate) and escalate — do NOT fall back to VLM-only (that's the unsafe design we're
    avoiding).
  - P2.B VLM null/timeout: NON-blocking — record vlm='timeout'/'blind', metrics decide the
    round, continue. Never retry a hung 32B (INV-7).
  - P2.C fixer null/!ok: exit the loop with best-so-far, mark needsUserJudgment (do not spin).
  - P3 export fails: retry once; if still failing, leave the best WIP glb + metrics on the
    server, mark needsUserJudgment, escalate.
  - Default if unspecified: pause and escalate.

### 12.6 Audit trail location
  Per-round evidence: renders/game/<anim>_v2.round<r>.metrics.json + the /tmp contact sheets
    referenced in critLog.
  Final rollup: renders/game/<anim>_v2.metrics.json (final metrics + critLog summary) +
    renders/game/<anim>_v2.mp4 (the human artifact).
  (No runtime/audits tree in this repo; evidence lives beside the render output, committed.)
